# PRD：AI 择校建议系统

> 版本：v0.1 | 日期：2026-04-11 | 状态：草稿

---

## 一、项目背景

文书老师（美国留学申请赛道）日常需要根据每位学生的背景，人工筛选目标院校并制定申请策略，耗时耗力。本项目旨在通过 AI 辅助，将"收集学生背景 → 生成择校建议"的流程产品化，帮助文书老师提升效率、扩大服务规模。

---

## 二、目标用户

| 角色 | 说明 |
|------|------|
| **文书老师（B 端）** | 系统购买方，通过后台生成一次性链接，发送给学生 |
| **学生（C 端）** | 通过专属链接进入页面，填写背景信息，获取 AI 择校建议 |

---

## 三、核心功能

### 3.1 一次性链接系统

- 文书老师在管理后台批量生成专属链接（每条链接绑定唯一 Token）
- 每条链接**仅限使用一次**：学生首次打开即标记为已使用，刷新或重新访问均不可再次填写（仅支持查看历史记录）
- 可选：链接支持设置有效期（如 7 天内有效）
- 链接格式示例：`https://xxx.com/apply?token=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`

### 3.2 学生信息收集表单

多步骤引导式表单，分步填写，减少填写压力：

**Step 1：基本信息**
- 姓名、年级、意向入学时间（学期 + 年份）

**Step 2：学术背景**
- GPA（当前 / 预估）及满分制
- 标化成绩：SAT / ACT（可选）
- 英语成绩：TOEFL / IELTS（可选）
- 就读学校类型（高中 / 本科）、专业（如为本科阶段）

**Step 3：意向信息**
- 目标专业 / 研究方向（可多选）
- 学位类型（本科 / 研究生）
- 意向州或地区（可选）
- 学校类型偏好：综合排名 / 专业排名 / 小而美文理学院等
- 预算范围（年学费，USD）

**Step 4：软实力补充**
- 课外活动、实习、科研、竞赛等（自由文本，500字以内）
- 申请亮点或特殊情况说明（可选）

### 3.3 AI 择校建议生成

- 学生提交表单后，系统将结构化数据注入预设 Prompt，调用 AI 接口生成建议
- 建议内容包含：
  - **冲刺校**（2~3 所）：录取难度高，但匹配意向
  - **主申校**（3~5 所）：录取概率合理，综合性价比高
  - **保底校**（2~3 所）：录取稳定，兜底选项
  - 每所学校附说明：推荐理由、专业优势、录取参考数据
- 结果页支持**流式输出**（打字机效果），提升体验
- 结果可导出为 PDF 或复制文本

### 3.4 数据存储

- 每次学生提交记录存入数据库（Token + 填写内容 + AI 建议 + 提交时间）
- 同步写入飞书多维表格，供文书老师直接查看和管理
- 数据字段设计见附录 A

### 3.5 管理后台（文书老师使用）

- 登录后可操作：
  - 批量生成链接（指定数量）
  - 查看链接状态：未使用 / 已使用 / 已过期
  - 查看每位学生的填写详情及 AI 建议
  - 导出数据（CSV）

## 四、用户流程

```
文书老师后台生成链接
        ↓
发送链接给学生（微信 / 邮件）
        ↓
学生打开链接 → 系统校验 Token 有效性
        ↓
Token 有效 → 进入多步骤表单填写
Token 无效/已使用 → 提示错误页
        ↓
学生提交表单 → Token 标记为已使用
        ↓
AI 实时生成择校建议（流式输出）
        ↓
学生查看结果 / 导出 PDF
        ↓
数据同步写入数据库 + 飞书多维表格
        ↓
文书老师在后台 / 飞书查看数据
```

---

## 五、非功能性需求

| 维度 | 要求 |
|------|------|
| **响应速度** | AI 开始输出首字符 ≤ 3 秒 |
| **可用性** | 移动端 + PC 端均可正常填写 |
| **安全性** | Token 一次性、不可预测（UUID v4）；后台需登录鉴权 |
| **数据保留** | 学生数据至少保留 1 年 |
| **并发** | MVP 阶段支持 20 并发用户同时填写 |

## 六、技术方案（推荐）

| 层次 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | Next.js + Tailwind CSS | 表单页 + 结果展示页 + 管理后台 |
| 后端 | Next.js API Routes | Token 校验、AI 调用、数据写入 |
| 数据库 | Supabase（PostgreSQL） | Token 管理 + 学生数据存储 |
| AI 接口 | Claude API（Anthropic） | 择校建议生成，支持流式输出 |
| 数据同步 | 飞书多维表格 API | 自动同步每条提交记录 |
| 部署 | Vercel | 极简运维，自动 CI/CD |

---

## 七、MVP 范围界定

**MVP 包含：**
- ✅ 一次性 Token 校验系统
- ✅ 多步骤表单（全字段）
- ✅ AI 择校建议生成（流式输出）
- ✅ 数据写入 Supabase + 同步飞书多维表格
- ✅ 简单管理后台（生成链接 + 查看数据）

**MVP 暂不包含：**
- ❌ 学生自助付款（由老师预购链接代替）
- ❌ 邮件自动发送链接
- ❌ PDF 导出
- ❌ 多语言支持

## 八、关键决策记录

| # | 决策 | 说明 |
|---|------|------|
| 1 | 表单语言：**中英双语可切换** | 用户可在表单顶部选择语言 |
| 2 | AI 建议语言：**与表单语言保持一致** | 若用户选中文表单，AI 输出中文建议 |
| 3 | 链接有效期：**30 天内有效，一次使用** | 超过 30 天自动过期；首次提交后 Token 立即标记为已使用 |
| 4 | 院校数据：**纯靠 AI 知识库** | 不预先导入结构化数据库，降低 MVP 复杂度 |
| 5 | 多账号：**MVP 仅支持单账号** | 先用固定密码登录管理后台，后续再扩展多账号权限 |
| 6 | 飞书表格：**创建在「我的文档库」** | 自动同步每条学生记录至飞书多维表格 |

## 附录 A：数据库字段设计（草案）

### tokens 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| token | VARCHAR | 唯一 Token 值 |
| status | ENUM | unused / used / expired |
| created_at | TIMESTAMP | 创建时间 |
| used_at | TIMESTAMP | 使用时间 |
| expires_at | TIMESTAMP | 过期时间（可选） |
| created_by | VARCHAR | 创建人（老师账号） |

### submissions 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| token_id | UUID | 关联 Token |
| student_name | VARCHAR | 学生姓名 |
| grade | VARCHAR | 年级 |
| gpa | DECIMAL | GPA |
| gpa_scale | DECIMAL | 满分制（4.0 / 5.0 等） |
| sat_score | INT | SAT 成绩（可空） |
| act_score | INT | ACT 成绩（可空） |
| toefl_score | INT | TOEFL 成绩（可空） |
| ielts_score | DECIMAL | IELTS 成绩（可空） |
| target_major | TEXT | 目标专业 |
| degree_type | VARCHAR | 本科 / 研究生 |
| budget_usd | INT | 年预算（美元） |
| extracurriculars | TEXT | 课外活动描述 |
| ai_suggestion | TEXT | AI 生成的建议全文 |
| submitted_at | TIMESTAMP | 提交时间 |
